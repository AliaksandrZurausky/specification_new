## Правила разработки

### Обязательно
- Всю кастомную разработку вести только в `/local/` — никогда в `/bitrix/`
- Использовать **ORM D7** (`\Bitrix\Main\ORM`) для работы с БД
- Использовать **namespace** в кастомных модулях
- Подключать модули через `Loader::includeModule('...')` перед использованием
- Работать с файловой системой через `\Bitrix\Main\IO\*`

### Запрещено
- Прямые SQL-запросы без ORM (исключение: сложные запросы, которые нельзя выразить через ORM)
- Изменять файлы в `/bitrix/`
- Использовать устаревший API (`CIBlockElement`, `CUser` и т.п.) в новом коде — только при крайней необходимости для совместимости

### Предпочтительный стиль
- PSR-4 автозагрузка в модулях
- Обработчики событий — через `EventManager` в `init.php` или в `module.php`
- Кеширование через `\Bitrix\Main\Data\Cache` / тегированный кеш `\Bitrix\Main\Data\TaggedCache`
